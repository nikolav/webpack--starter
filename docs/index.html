#package.json

- includes standard modern js build stack, babel, webpack, eslint, prettier. css gets processed with postcss, sass and tailwindcss included as well.


#App

- loads App class and css in /src

- simulates reactjs concepts by maintaining local state and syncing ui based on state updates. `app.e` field is simple event-listener implementation for registering and triggering events.


#render

- gets called whenever `.setState` is called to render new ui based on updated state. it basically renders page html implementing simple router based on state flag `.active` which is current page selected through ui.


#pages, and assets

- pages are implemented as 'component methods' and return html to render. icons are implemented as methods on App and return svgs to render. images are not used but ive configured weback to recognize that sort of asset.


#calculations

- for running tax calculation ive used retail rates in small sale..


#styles, css

- for styling used normalize reset, tailwind, some scss. tailwind gets exorted in /styles and folder gets imported in /src/index.js.


#color mode

- for dark mode ive used tailwind `.class` option to namespace classes under .documentElement, and use dark: variant to apply in styles. when <html> gets class="dark" color mode classes get applied. implemented simple color-mode class for switching dark mode state, which gets synced with user 'prefers-color-mode' preference. methods .setDark .setLight toggle class .dark and set localStorage flag to set color-mode on first page load.


#chart

- draw pie chart with d3. it takes data array with key-value pairs [{ [key: string]: value: number }]; function .chartDonut takes Node to render svg in, data[], and aditional options for fine-tuning.


#pdf download

- send post request with json calculation to php script @heroku which uses dompdf laravel plugin to generate pdfs from blade views. file takes json input .calculation and loads blade view to render pdf from.


#utils

- exported global utility methods from /src/util
